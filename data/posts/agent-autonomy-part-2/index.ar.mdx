بـ[الجزء الأول](/free-writing/agent-autonomy)، حكيت إنه المسائل المحدودة اللي إلها feedback loop واضح مثالية للـ agents المستقلة. واستخدمت مسألة ترتيب كمثال. هلق خلّينا نغوص بالخوارزمية الحقيقية.

## المسألة، بالظبط

عندك N عنصر. بدك ترتّبهم. بس دالة المقارنة عندك *عليها noise*: لما تسأل "هل A أكبر من B؟"، بترجّعلك جواب غلط بنسبة معينة من المرات.

خوارزميات الترتيب العادية بتفترض إنه المقارنات موثوقة. merge sort، quicksort—كلها بتنهار لما المقارنات تكذب.

السؤال: بتقدر ترتّب بشكل موثوق لما تكون مقارناتك مش موثوقة؟

## المقاربة الساذجة

الحل الواضح: كرّر كل مقارنة كذا مرة وخد تصويت بالأغلبية.

إذا قارنت A وB خمس مرات وA فاز تلاتة، يعني A على الأغلب أكبر. هاد بيزبط، بس مكلف—هلق عم تعمل 5 أضعاف المقارنات.

وكمان ما بيتكيّف. في مقارنات سهلة (عناصر واضح إنها مختلفة)، وفي صعبة (عناصر متقاربة كتير). إنك تصرف خمس مقارنات على حالة واضحة هدر جهد.

## فكرة الـ Agent

الـ agent اكتشف اشي شاطر: **كيّف عدد المقارنات حسب الثقة.**

لكل زوج، ضلّ قارن لحد ما تصير واثق كفاية من النتيجة. الأزواج السهلة بتنحل بمقارنة أو تنتين. الصعبة بتاخد اهتمام أكتر.

هاد بالأساس sequential probability ratio test (SPRT) مطبّق على الترتيب.

## كيف بيشتغل

لكل مقارنة، تابع عدّاد الفوز/الخسارة. احتفظ بتقدير جاري لاحتمال إنه A > B. ووقّف لما الاحتمال يتعدّى عتبة ثقة.

```python
def confident_compare(a, b, noisy_compare, threshold=0.95):
    wins_a, wins_b = 0, 0
    while True:
        if noisy_compare(a, b):
            wins_a += 1
        else:
            wins_b += 1
        # Compute confidence using beta distribution
        total = wins_a + wins_b
        confidence = beta_confidence(wins_a, total)
        if confidence > threshold:
            return a if wins_a > wins_b else b
        if total >= MAX_COMPARISONS:  # safety bound
            return a if wins_a > wins_b else b
```

الأناقة: بيصرف المقارنات وين بتفرق. الحالات الواضحة بتنحل بسرعة. الحالات الملتبسة بتاخد الاهتمام اللي بدها ياه.

## النتائج

مقارنةً بالتصويت بالأغلبية بتكرار ثابت:
- **40% مقارنات أقل** لنفس الدقة
- سلوك تكيّفي طلع طبيعي من عتبة الثقة
- أسوأ حالة محدودة بـ MAX_COMPARISONS

الـ agent ما طبّق بس خوارزمية معروفة—هو اشتقّ المقاربة من المبادئ الأولى، جرّبها، وأكّد التحسّن.

## شو بتعلّمنا

أبعد من الخوارزمية بحد ذاتها، في درس أعمق هون عن المقارنات اللي عليها noise بشكل عام:

كل ما تكون عم تجمّع إشارات عليها noise—feedback من المستخدمين، A/B tests، مخرجات الموديل—نفس المبدأ بنطبّق. **اصرف ميزانية العيّنات عندك وين الغموض أعلى.**

هاي الفكرة الأساسية ورا الـ multi-armed bandits، الـ active learning، والاختبار التسلسلي. الـ agent أعاد اكتشاف مبدأ أساسي بالإحصاء بس لأنه حاول يحل مسألة محدودة منيح.

## الخلاصة

أعطِ agent شاطر مسألة محدودة بـ feedback واضح، ومش بس رح يحلها—يمكن يعلّمك اشي. حل الـ noisy sorting منيح فعلاً—كنت بنزّله بدون ما أفكّر مرتين.

هاد وعد الاستقلالية على المسائل الصح: مش بس أتمتة، كمان اكتشاف.
